
## Что сделал

Сегодня я добавил `TransactionResult` для обработки результата транзакции.

Раньше `ProcessTransaction()` просто менял внутренний статус транзакции. Теперь метод возвращает отдельный объект результата, который содержит:

- transactionId;
- senderId;
- recipientId;
- amount;
- status;
- message.

## Зачем это нужно

`TransactionResult` нужен, чтобы операция не просто меняла состояние внутри объекта, а возвращала наружу понятный результат.

Это ближе к backend-подходу, потому что такой результат потом можно:

- вывести в консоль;
- записать в лог;
- вернуть из API;
- сохранить в базу данных;
- использовать в тестах.

## Что изменилось в логике

`ProcessTransaction()` теперь возвращает результат для разных сценариев:

- успешная транзакция → `Completed`;
- повторная обработка → сообщение `Transaction already processed`;
- недостаточно средств → `InsufficientFunds`;
- некорректная сумма → `InvalidAmount`;
- некорректные участники → `InvalidParticipants`.

## Что я понял

Я понял, что статус внутри объекта и результат операции — это не одно и то же.

Статус показывает текущее состояние транзакции.

Результат операции показывает, что произошло при конкретном вызове `ProcessTransaction()`.

Например, повторный вызов уже завершённой транзакции может вернуть сообщение `Transaction already processed`, но сам статус транзакции останется `Completed`.

## Проверенные сценарии

1. Перевод 30 от `agent1` к `agent2`.
    - Ожидаемо: `Completed`.
    - Балансы изменились.
2. Повторный вызов `ProcessTransaction()` для той же транзакции.
    - Ожидаемо: деньги не переводятся второй раз.
    - Сообщение: `Transaction already processed`.
3. Перевод 10000.
    - Ожидаемо: `InsufficientFunds`.
    - Балансы не изменились.
4. Перевод 0.
    - Ожидаемо: `InvalidAmount`.
    - Балансы не изменились.
