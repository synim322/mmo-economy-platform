
## Назначение

Domain Model описывает основные сущности проекта MMO Economy Platform и их ответственность.

Проект симулирует экономику онлайн-игры. Главная цель модели — не просто хранить данные, а защищать правила экономики.

## Wallet

`Wallet` отвечает за деньги агента.

Ответственность:

- хранит баланс;
- не позволяет балансу стать отрицательным;
- проверяет корректность суммы;
- выполняет пополнение;
- выполняет списание;
- сообщает результат операции.

Инварианты:

- баланс не может быть меньше нуля;
- нельзя пополнить кошелёк некорректной суммой;
- нельзя списать некорректную сумму;
- нельзя списать больше, чем есть на балансе.

## Agent

`Agent` представляет участника экономики.

Это может быть NPC, игрок или бот симуляции внутри нашей собственной системы.

Ответственность:

- хранит id;
- хранит имя;
- владеет кошельком;
- делегирует операции с деньгами в `Wallet`.

Инварианты:

- id должен быть валидным;
- имя не должно быть пустым;
- агент всегда должен иметь кошелёк.

## Почему Agent не проверяет баланс сам

Проверка баланса находится в `Wallet`, потому что именно кошелёк отвечает за деньги.

Если проверку баланса размазать по `Agent`, `Transaction`, `Market` и другим классам, правила экономики станут хрупкими. Один класс может проверить одно правило, другой класс — забыть. Это приведёт к багам.

Поэтому правило такое:

`Wallet` защищает баланс.  
`Agent` владеет кошельком.  
`Transaction` будет описывать факт или попытку перевода между агентами.

## Следующая сущность: Transaction

`Transaction` нужна, чтобы перевод денег был не просто двумя вызовами методов, а отдельной доменной операцией.

Она должна помочь ответить на вопросы:

- кто отправитель;
- кто получатель;
- какая сумма;
- прошла операция или нет;
- почему операция могла быть отклонена.
